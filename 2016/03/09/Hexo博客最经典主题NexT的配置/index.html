Hexo博客最经典主题NexT的配置

发表于 2016-03-09   |   分类于 Hexo   |     |   阅读次数

主要介绍NexT主题官网没有介绍到的地方,其余基本配置请参考NexT主题原站.太困了,半夜三点了,下面直接进入正题.

复制NexT的主题

以下进入复制主题环节，如果那一步出现问题，或者修改后没有显示修改的结果，建议来来一个，再看看，可以解决很多问题。

1. $ hexo clean
2. $ hexo g
3. $ hexo s

建立了Hexo文件之后就可以复制主题了.
输入指令$ git clone https://github.com/iissnan/hexo-theme-next themes/next开始复制NexT主题,如图所示:

启用NexT的主题

用文本编辑器修改Hexo目录下的config.yml配置文件中的theme属性，将其设置为next
theme: next

注意:字段名和值之间是有空格的,后面所有的字段名都一样

如图所示:

注意：Hexo有两个config.yml文件，一个在根目录，一个在theme下，此时修改的是在根目录下的。

验证主题是否启用

运行 hexo s --debug，并访问 http://localhost:4000，确保站点正确运行。


或者$ hexo s启动本地服务，进行文章预览调试.这只是调试,还记得上篇文章说的每次修改本地文件后，需要hexo generate才能保存.

常用的指令如下: 

1. hexo g == hexo generate(生成)
2. hexo d == hexo deploy(部署)
3. hexo s == hexo server(启动本地服务器)

修改NexT主题

我这里只简单的补充一些没有涉及到的.其他的请参考官方指导,有问题可以M我.

显示博客的阅读次数

因为Next主题已经集成了相关代码，所以直接部署就生效了要注册LeanCloud帐号,其他主题请参考这篇文章.

配置 LeanCloud

点这里注册,注册完成LeanCloud帐号并验证邮箱之后，我们就可以登录我们的LeanCloud帐号，进行一番配置之后拿到 AppID 以及 AppKey 这两个参数即可正常使用文章阅读量统计的功能了。

创建应用

我们新建一个应用来专门进行博客的访问统计的数据操作。首先，打开控制台，如下图所示：

在出现的界面点击”创建应用” ：

在接下来的页面，新应用的名称我们可以随意输入，即便是输入的不满意我们后续也是可以更改的:

这里为了演示的方便，我新创建一个取名为test的应用。创建完成之后我们点击新创建的应用的名字来进行该应用的参数配置：

创建Class

在应用的数据配置界面，左侧下划线开头的都是系统预定义好的表，为了便于区分我们新建一张表来保存我们的数据。点击左侧右上角的齿轮图标，新建Class：

在弹出的选项中选择 创建Class 来新建Class用来专门保存我们博客的文章访问量等数据:
点击创建Class之后，此处的新建Class名字 必须 为Counter:

提取AppID和AppKey

创建完成之后，左侧数据栏应该会多出一栏名为Counter的栏目，这个时候我们点击顶部的设置，切换到test应用的操作界面:

在弹出的界面中，选择左侧的 应用Key 选项，即可发现我们创建应用的 AppID 以及 AppKey ，有了它，我们就有权限能够通过主题中配置好的Javascript代码与这个应用的Counter表进行数据存取操作了:

修改主题配置文件

复制 AppID 以及 AppKey 并在NexT主题的 _config.yml(在F:\Hexo\themes\next目录下) 文件中我们相应的位置填入即可，正确配置之后文件内容像这个样子: 

1. leancloud_visitors:
2.   enable: true
3.   app_id: FA8WROigvdCXWOhG3eKWz32r-gzGzoHsz
4.   app_key: 4PD0IXceSTXeWf8U3AWtUWqK

这个时候重新生成部署Hexo博客，应该就可以正常使用文章阅读量统计的功能了。需要特别说明的是：记录文章访问量的唯一标识符是文章的 发布日期 以及 文章的标题 ，因此请确保这两个数值组合的唯一性，如果你更改了这两个数值，会造成文章阅读数值的清零重计。

Hexo写博客的常用命令:

标题

$ hexo new title (PS:title代表你想起的文件名,会自动在F:\Hexo\source_posts目录下生成title.md文件)

显示全文

博客默认显示全文,推荐在不需要显示的地方输入 <!-- more --> 进行手动进行截断.

用Hexo发表文章的Markdown语法

1. title: XXXXXX #文章页面上的显示名称，可以任意修改，不会出现在URL中
2. date: 2016-03-09 03:30:16 #文章生成时间，一般不改，当然也可以任意修改
3. categories: example #分类
4. tags: [tag1,tag2,tag3] #文章标签，可空，多标签请用此格式，所有的标签写在中括号里面,标签之间用逗号隔开

将独立域名与GitHub的空间绑定

GitHub的设置

方法一：在Repository的根目录下面，新建一个名为CNAME的文本文件，里面写入你要绑定的域名，比如byebyebye.xyz(这个不会)

方法二：在F:\Hexo\source下新建一个文本文件,内容为:byebyebye.xyz,保存为CNAME文件(无后缀名)，用hexo命令$ hexo d -g提交上去。
