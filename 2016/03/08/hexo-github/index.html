如何利用hexo-github搭建个人博客

发表于 2016-03-08 | 分类于 Hexo | | 阅读次数:

        最新教程,截止2016年3月8日 23:56:03.参考了众多文章,总结出来的,纠正了一些错误,希望能帮助大家少走弯路.开始之前,简单的介绍下Hexo,其实也就一句话:Hexo是一个快速、简洁且高效的博客框架。使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。之所以选择github,一方面是因为有免费的空间,省去了租服务器的费用,只需要注册个域名就好,如果不需要顶级域名,完全可以免费着来;另一方面它是个不错的开源平台,大牛超级多.多余的话不多说了,下面进入正题.

安装准备软件

首先下载安装node.js和Git。

  1. Node.js
  2. Git

PS:安装路径没必要安装在C盘

打开Git的方法

  1. 开始菜单–Git Bash

  2. 鼠标右键打开Git Bash

配置和使用Github

配置SSH keys

        我们如何让本地git项目与远程的github建立联系呢？用SSH keys。

  • 检查SSH keys的设置
    首先我们需要检查你电脑上是否有ssh key.打开git输入代码:$ cd ~/.ssh检查本机的ssh密钥.
    如果出现如下图提示：No such file or directory 说明你是第一次使用git。跳到第三步，否则继续。
  • 备份和移除原来的ssh key设置
    因为已经存在key文件，所以需要备份旧的数据并删除 
1.    $ ls
2.    config  id_rsa  id_rsa.pub  known_hosts
3.    $ mkdir key_backup
4.    $ cp id_rsa* key_backup
5.    $ rm id_rsa*
  • 生成新的SSH Key
    输入代码 $ ssh-keygen -t rsa -C "邮件地址@youremail.com" 后,就会弹出下面的代码，就生成了新的key文件，我们只需要默认设置就好，所以当需要输入文件名的时候，回车就好。 
    1.    $ ssh-keygen -t rsa -C "邮件地址@youremail.com"
2.    Generating public/private rsa key pair.
3.    Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

如下图所示:

PS:1:此处的邮箱地址，你可以输入自己的邮箱地址；2:此处的「C」的是大写的「C」
然后系统会要你输入加密串（Passphrase）,如下图所示:

PS:输入密码时候是不显示的，你直接输入就可以了。
回车后,就会看到如下的界面, 就成功设置ssh key了.

  • 添加SSH Key到GitHub
    在本机设置SSH Key之后，需要添加到GitHub上，以完成SSH链接的设置。

  1. 用文本编辑器打开id_rsa.pub 这个文件路径是C:\Users\lxd\.ssh\id_rsa.pub(每个人的路径可能不一样,具体路径在生成SSH Key的时候会出现),此文件里面内容为刚才生成的密钥。如果看不到这个文件，你需要设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功.

  2. 登陆github系统。点击右上角的头像—>Setting—>SSH keys —> New SSH key.

  3. 把你本地生成的密钥复制到里面（key文本框中）,点击 Add SSH key 就ok了.具体图示如下:


  • 测试一下
    可以输入下面的命令，看看设置是否成功，git@github.com的部分不要修改：
    > $ ssh -T git@github.com

如果是下面的反应：

The authenticity of host 'github.com (192.30.252.130)' can't be established.
RSA key fingerprint is SHA256:nThbg6kXUpJWGl7E1IGOCspRomTxdCARLviKw6E5SY8.
Are you sure you want to continue connecting (yes/no)?

不要紧张，输入yes就好，然后会看到如下代码提示你输入密码(就是刚才生成SSH时设置的)：

Enter passphrase for key '/c/Users/lxd/.ssh/id_rsa':

如下图:


输入密码之后,就会看到如下提示:

Hi MinhowS! You've successfully authenticated, but GitHub does not provide shell access.

  • 设置你的账号信息

现在你已经可以通过SSH链接到GitHub了，还有一些个人信息需要完善的。
Git会根据用户的名字和邮箱来记录提交。GitHub也是用这些信息来做权限的处理，输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己的，名字必须是你的真名，而不是GitHub的昵称。(搞不懂真名是几个意思,反正我填的是Github的昵称)

1. $ git config --global user.name "minhow"//用户名   最后我改成MinhowS了
2. $ git config --global user.email  "你的邮箱名@hotmail.com"//填写自己的邮箱

如下图:

  • SSH Key配置成功
    本机已成功连接到github。

使用GitHub Pages建立博客

与GitHub建立好链接之后，就可以方便的使用它提供的Pages服务，GitHub Pages分两种，一种是你的GitHub用户名建立的username.github.io这样的用户&组织页（站），另一种是依附项目的pages。想建立个人博客是用的第一种，形如MinhowS.github.io这样的可访问的站，每个用户名下面只能建立一个。

  • Github上建立仓库

登录后系统，在github首页，点击页面右上角的”+”,之后选择New Repository,如下图所示:

  • 填写项目信息

project name：MinhowS.github.io
description：填写这个项目的描述信息

(PS:Github Pages的Repository名字是特定的，比如我的Github账号是MinhowS，那么我的Github Pages Repository名字就是MinhowS.github.io。点击「Create Repository」 完成创建。)

搭建Hexo环境

Hexo介绍

Hexo的作者是tommy351，根据官方介绍，Hexo是一个简单、快速、强大的博客发布工具，支持Markdown格式。

安装Hexo

打开git。输入命令:$ npm install -g hexo,即可完成安装.默认安装到node.js的目录下.

部署Hexo

在我的电脑中建立一个名字叫「Hexo」的文件夹(我的路径F:\Hexo)，然后在此文件夹中右键打开Git Bash。 在弹出的命令窗口中输入代码$ hexo init,Hexo随后会自动在目标文件夹建立网站所需要的所有文件。如图所示:


用文本编辑器编辑_config.yml(在F:\Hexo下), 翻到最下面，改成我这样子的.在你部署时,要把MinhowS改成你的帐号名.

  1. deploy:
  2. type: git
  3. repository: https://github.com/MinhowS/MinhowS.github.io.git
  4. branch: master

如图所示:


然后输入代码npm install hexo-deployer-git --save安装插件,如图所示:

PS:有的网友在安装时出现如下错误:

  1. npm ERR! Darwin 15.2.0
  2. npm ERR! argv “/usr/local/bin/node” “/usr/local/bin/npm” “install” “hexo-deployer-git” “–save”
  3. npm ERR! node v4.2.4
  4. npm ERR! npm v2.14.12
  5. npm ERR! shasum check failed for /var/folders/n4/m1w5r2rj3y77vpn_s6g87s180000gn/T/npm-8320-c250e6be/registry.npmjs.org/moment/-/moment-2.11.0.tgz
  6. npm ERR! Expected: 584f5fb87c1f637f12937edce127dde2d206d705
  7. npm ERR! Actual: f3f4cb5f46c790b29b9b7bdce8435fe70ad58dbd
  8. npm ERR! From: https://registry.npmjs.org/moment/-/moment-2.11.0.tgz

先用npm install hexo-deployer-git安装，再执行npm install hexo-deployer-git --save 即可.

记住：每次修改本地文件后，需要hexo generate才能保存。每次使用命令时，都要在F:\hexo目录下(这是我部署Hexo时的路径,这里注意要是你们自己的)。由于搭建环境时自带默认主题,浏览器输入http://localhost:4000，即可查看搭建效果。
